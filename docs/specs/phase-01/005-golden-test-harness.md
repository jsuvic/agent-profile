# Spec: Golden Test Harness

## Status

Verified

Implemented in `packages/compiler` for byte-for-byte comparison of the minimal
fixture generated outputs, including hidden generated paths. Verified on
2026-05-02 after final implementation review.

## Problem

Generated files are public product contracts. Without golden tests, template
changes can silently alter outputs and break users' agent instructions,
lockfiles, or safety posture.

## Goal

Define a golden test harness for compiler outputs.

## Non-Goals

- approving any specific generated output content
- adding visual snapshots
- adding hosted test services
- auto-updating fixtures without review
- replacing unit tests for schema validation or compiler helpers

## User Flow

1. A developer changes a compiler target, template, or deterministic formatting
   helper.
2. Golden tests read fixture profiles and expected files.
3. Golden tests run the compiler in memory.
4. Generated bytes are compared byte-for-byte to expected files.
5. If output changes, tests fail with path-specific diffs.
6. The developer updates fixtures only when an approved spec requires the output
   change.

## Inputs

- fixture profiles
- expected generated output files
- deterministic compiler API from `003-compiler-determinism.md`
- target specs such as `004-agents-md-target.md`

## Outputs

- pass/fail golden test report
- stable path-specific failure details
- readable unified diff for changed generated files

## Fixture Layout

Canonical fixture layout:

```text
fixtures/<fixture-name>/
  ai-profile.yaml
  expected/
    AGENTS.md
    ai-profile.lock
```

Nested generated paths map directly under `expected/`:

```text
generated path: .codex/config.toml
expected path:  fixtures/<fixture-name>/expected/.codex/config.toml
```

Hidden expected files must be included in traversal.

## Failure Modes

```ts
type GoldenFailure = {
  code:
    | "fixture_profile_invalid"
    | "compiler_error"
    | "missing_expected_file"
    | "extra_expected_file"
    | "content_mismatch";
  fixture: string;
  path: string;
  message: string;
};
```

Rules:

- `fixture` is the fixture directory name.
- `path` is the generated repository-relative path.
- Failures are sorted by `fixture`, then `path`, then `code`.
- `content_mismatch` includes a unified diff outside this object.
- Messages must not include secret values or environment variable values.

## Diff Rules

- Diffs are unified diffs.
- Diff paths use repository-relative generated paths.
- Diff content is normalized to LF for display.
- Binary generated files are out of scope for the MVP.
- Golden tests compare exact bytes, even though diff display normalizes for
  readability.

## Contracts

- Expected files live under `fixtures/<fixture-name>/expected/`.
- Fixture profile inputs live at `fixtures/<fixture-name>/ai-profile.yaml`.
- Golden comparisons are byte-for-byte.
- Fixture traversal order is lexicographic by fixture name.
- Expected file traversal order is lexicographic by generated path.
- Generated output paths map directly to expected paths under `expected/`.
- Missing expected output fails.
- Extra expected output fails.
- Fixture updates require a spec-backed reason.
- Golden tests must not mutate fixtures.

## Security Rules

- Golden tests must not read ignored secret files.
- Golden tests must not upload fixture content.
- Golden tests must not execute shell commands beyond the test runner.
- Golden tests must not install dependencies.
- Fixture files must not contain literal secrets.
- Failure messages must not print environment variable values.

## Acceptance Criteria

- Golden test directory layout is documented.
- Generated path to expected path mapping is explicit.
- Golden comparison behavior is byte-for-byte.
- Fixture and expected file ordering are deterministic.
- Missing and extra expected files fail clearly.
- Output diffs are actionable and path-specific.
- Fixture update rules are documented.

## Tests

- passing golden fixture test
- failing content mismatch produces `content_mismatch`
- failing content mismatch includes readable unified diff
- missing expected output produces `missing_expected_file`
- extra expected output produces `extra_expected_file`
- invalid fixture profile produces `fixture_profile_invalid`
- compiler failure produces `compiler_error`
- fixture ordering is stable
- expected file ordering is stable
- hidden expected files are included
- golden tests do not mutate fixtures
- failure output does not include secret-like values
