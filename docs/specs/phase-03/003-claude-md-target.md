# Spec: CLAUDE.md Target

## Status

Verified

Verified on 2026-05-03 after compiler implementation, golden fixture update,
and compiler test run. Official Claude Code memory and skills docs were
re-checked before implementation. Earlier approval added a
compiler-determinism input cross-reference and an explicit non-goal for
`./.claude/CLAUDE.md`.

## Problem

The compiler currently generates shared `AGENTS.md`, Claude settings, and
Claude MCP configuration, but it does not yet generate the Claude project
instruction file. The MVP plan still calls for a Claude-specific instruction
target that avoids duplicating the full `AGENTS.md` body while making Claude
Code load the shared project rules.

## Goal

Generate a deterministic repository-root `CLAUDE.md` file for the `claude-md`
target when `clients.claude.enabled: true`.

The generated file imports `AGENTS.md` and then adds a short Claude-specific
delta for memory scope, generated Claude settings, workflow skills, and runtime
verification limits.

## Non-Goals

- implementing the target
- generating Claude workflow skills
- generating `.claude/rules/`
- generating `./.claude/CLAUDE.md` (the target emits the repo-root file only)
- generating user-level or managed-policy Claude memory
- generating `CLAUDE.local.md`
- enabling or disabling Claude auto memory
- changing live Claude runtime flags, user settings, or managed settings
- replacing `AGENTS.md` as the shared cross-client instruction source

## User Flow

1. A user enables `clients.claude.enabled: true`.
2. A future compile command renders `CLAUDE.md` as a Claude-specific project
   instruction target.
3. The generated `CLAUDE.md` imports generated `AGENTS.md` with `@AGENTS.md`.
4. Claude Code loads `AGENTS.md` plus the Claude-specific delta at session
   start.
5. Golden tests compare the generated file byte-for-byte.

## Generated Artifact Shape

For `fixtures/minimal-valid/ai-profile.yaml`, the exact golden output must be:

```markdown
<!-- Generated by Agent Profile Compiler. Do not edit by hand. -->

@AGENTS.md

# Claude Code

## Source Of Truth

The shared project rules are imported from `AGENTS.md` above. This file only adds Claude-specific guidance.

## Runtime Scope

- Treat `CLAUDE.md` as behavioral context, not as an enforcement layer.
- Use generated `.claude/settings.json` and `.mcp.json` as the intended project configuration when present.
- Report user settings, local settings, managed settings, CLI flags, and session mode changes as not verifiable unless they are actually inspected.
- Keep repeatable SDD, TDD, and final-review procedures in generated `.claude/skills/` files instead of expanding them here.

## Safety Rules

- Do not upload source code.
- Do not read or print secrets.
- Do not write literal tokens into generated files.
- Do not grant production access.
- Do not install dependencies automatically.
- Ask before mutating files, running shell commands, installing dependencies, or using external network access.
```

`@AGENTS.md` is intentionally an import, not a plain text reference. If
official Claude Code docs change import syntax or project instruction discovery,
update this spec and golden fixture before implementation.

## Target Mapping

Verified Claude Code memory surface for this spec revision:

Verification date: 2026-05-03.

- Claude Code uses `CLAUDE.md` files as persistent project/user/org
  instructions.
- Project instructions may live at `./CLAUDE.md` or `./.claude/CLAUDE.md`.
- `CLAUDE.md` can import additional files using `@path/to/import`.

  use `AGENTS.md` should import it from `CLAUDE.md`.

  procedures into skills or path-scoped rules.
- `CLAUDE.md` files in the directory hierarchy are loaded in full at launch;
  subdirectory files can load on demand.

## Contracts

- The target must emit exactly `CLAUDE.md`.
- The template id is exactly `targets/claude-md@1`.
- The generated file must import `AGENTS.md` using `@AGENTS.md`.
- The generated file must remain a short Claude-specific delta.
- The generated file must not duplicate the full `AGENTS.md` body.
- The generated file must not claim runtime enforcement beyond context
  guidance.
- The generated file must reference generated `.claude/settings.json` and
  `.mcp.json` as intended project config only.
- The generated file must not generate or request local/user/global memory
  writes.
- Golden tests compare exact output bytes.

## Security Rules

- Do not include literal secrets.
- Do not include environment variable values.
- Do not instruct Claude to upload source code.
- Do not grant production access.
- Do not instruct automatic dependency installation.
- Do not instruct unsafe auto-approval.
- Do not use imports outside the repository for generated project output.
- Do not generate `CLAUDE.local.md` or user-level memory.
