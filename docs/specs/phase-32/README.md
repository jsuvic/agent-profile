# Phase 32: Guided Repository Update

Phase 32 makes repository updates informed and non-destructive: valid custom
future configuration is preserved as user-owned, capability adoption becomes
an editable explained review, and supported YAML no longer falls into an opaque
unsafe-insertion refusal.

- Spec: [`001-guided-repository-update.md`](001-guided-repository-update.md)
- Issue briefs: [`issues/`](issues/)

The spec and governing ADR decisions were approved on 2026-07-14. Phase 32 is
sequenced after Phase 31 so the update journey can reuse the completed
permission lifecycle instead of introducing temporary manual guidance.
