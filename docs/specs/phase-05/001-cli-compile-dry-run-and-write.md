# Spec: CLI Compile Dry Run and Write

## Status

Verified

Approved for Phase 5 implementation on 2026-05-02. Implemented on
2026-05-03. Verified on 2026-05-03 with workspace checks, tests, and build.

## Problem

The compiler can generate deterministic files in memory, but users do not yet
have a CLI command to preview or write those files into a repository.

## Goal

Add `agent-profile compile` with dry-run output, explicit write mode, lockfile
generation, and diff-before-write safety.

## Non-Goals

- initializing a new profile
- importing existing artifacts
- stack detection
- automatic dependency installation
- network access
- changing client runtime settings
- generating hooks, subagents, plugins, global memory writes, or dedicated
  knowledge MCP/tool artifacts

## User Flow

```bash
agent-profile compile --root . --profile ai-profile.yaml --dry-run
agent-profile compile --root . --profile ai-profile.yaml --target agents-md --dry-run
agent-profile compile --root . --profile ai-profile.yaml --write
```

Dry-run is the default unless `--write` is provided.

## Inputs

- `--root <path>`, default `.`
- `--profile <path>`, default `ai-profile.yaml`, resolved relative to `--root`
- `--target <target-id>`, repeatable
- `--dry-run`
- `--write`
- `--force`, required for first-write protection cases
- validated `AiProfile`
- compiler output descriptors

## Outputs

- dry-run report listing files that would be created, changed, or unchanged
- optional file writes for generated artifacts
- optional `ai-profile.lock` write
- CLI exit code

Exit codes:

| Code | Meaning                                                            |
| ---- | ------------------------------------------------------------------ |
| `0`  | dry-run or write completed without errors                          |
| `1`  | profile validation, compiler, target, or write-safety error        |
| `2`  | argument parsing failure                                           |
| `3`  | write would replace protected existing files and `--force` missing |

## Contracts

- Dry-run must not write files.
- Write mode must use diff-before-write from
  `003-diff-before-write.md`.
- `--dry-run` and `--write` together are an argument error and exit `2`.
- `--profile` must be repository-relative under `--root` and must pass the same
  path validation boundary as other write/read paths.
- Unknown targets and disabled targets are compile errors and exit `1`.
- Generated bytes must come from the verified compiler API.
- `ai-profile.lock` must be generated from the same profile bytes, template
  descriptors, and generated file descriptors as the output files.
- The lockfile write is a planned write descriptor handled by
  `003-diff-before-write.md`, not a separate write path.
- Output ordering must be deterministic by path then target.
- CLI must not read files outside the selected profile, existing output paths,
  and lockfile path.
- Compile never deletes orphaned generated files. A future prune mode requires a
  separate approved spec.
- If a previous compile produced files that are no longer in the current
  compile result, compile leaves those files in place and removes them from the
  new lockfile. Doctor compares the current generated set and must not be
  treated as a prune command.
- Compile does not run doctor automatically; users invoke
  `agent-profile doctor` separately.
- First-write protection: when a generated output path already exists and no
  `ai-profile.lock` exists, or when the existing file hash does not match the
  lockfile-recorded hash for that path, `--write` must refuse to replace the
  file unless `--force` is also present.
- All output and lockfile paths must round-trip through `safeOutputPath` from
  `packages/compiler/src/shared.ts`.
- Compile only generates capabilities covered by approved target specs.
  Advanced capability requests from future schema work must produce explicit
  unsupported or not-generated messages until target-specific specs exist.

## Security Rules

- Do not upload profile or source contents.
- Do not read secret files.
- Do not print generated file contents unless a future explicit diff flag allows
  it.
- Do not install dependencies.
- Do not execute shell commands.
- Do not write unless `--write` is present.
- Do not silently generate advanced capability artifacts such as hooks,
  subagents, plugins, or global memory writes.

## Acceptance Criteria

- `agent-profile compile --dry-run` previews deterministic output paths.
- `agent-profile compile --write` writes generated files and `ai-profile.lock`.
- `--target` limits generation to selected targets.
- Unknown target exits non-zero with a deterministic error.
- Disabled target exits non-zero with a deterministic error.
- Dry-run leaves the repository unchanged.
- Write mode refuses unsafe output paths.
- Written files match golden/compiler bytes exactly.
- Second `--write` invocation produces identical generated bytes and identical
  lockfile hashes as the first.
- Existing user-authored generated-path files require `--force` when no
  matching lockfile entry proves they are compiler-owned.

## Tests

- dry-run reports create/change/unchanged without writing
- write creates expected files in a temp project
- write creates `ai-profile.lock`
- repeated write is idempotent
- selected target writes only selected outputs and lockfile
- unknown target exits non-zero
- disabled target exits non-zero
- `--dry-run --write` exits `2`
- protected first write without `--force` exits `3`
- lockfile and outputs are produced from the same compile result
- second write is byte-for-byte identical to the first
- generated bytes match compiler descriptors
