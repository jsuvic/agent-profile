# Spec: Import Existing Artifacts

## Status

Verified

Approved for Phase 5 implementation on 2026-05-02. Implemented on
2026-05-03. Verified on 2026-05-03 with workspace checks, tests, and build.

## Problem

Many repositories already have `AGENTS.md`, `CLAUDE.md`, or Tabnine guideline
files. Init should avoid overwriting them blindly and should help users compare
existing artifacts with generated intent.

## Goal

Add import analysis for existing supported agent artifacts.

## Non-Goals

- automatic semantic migration
- deleting existing files
- rewriting existing artifacts without explicit write mode
- importing unsupported clients
- importing hooks, subagents, plugins, global memory writes, or dedicated
  knowledge MCP/tool artifacts
- uploading files for analysis
- importing Cursor, Aider, Copilot, or other non-MVP client artifacts

## User Flow

```bash
agent-profile init --import --dry-run
agent-profile init --import --write
```

The command reports existing artifacts and suggests profile fields or manual
review actions. Any write still goes through diff-before-write.

## Inputs

Supported existing artifact paths:

- `AGENTS.md`
- `CLAUDE.md`
- `.tabnine/guidelines/*.md`
- `.tabnine/mcp_servers.json`
- `.codex/config.toml`
- `.claude/settings.json`
- `.mcp.json`

## Outputs

- deterministic import report
- optional generated `ai-profile.yaml` proposal
- warnings for unsupported or unsafe content

Import mapping:

| Existing artifact           | Profile effect                                                       |
| --------------------------- | -------------------------------------------------------------------- |
| `AGENTS.md`                 | no direct profile fields; report presence for manual review          |
| `CLAUDE.md`                 | `clients.claude.enabled = true`; report manual review                |
| `.tabnine/guidelines/*.md`  | `clients.tabnine.enabled = true`; report manual review               |
| `.tabnine/mcp_servers.json` | `clients.tabnine.enabled = true`; flag non-empty servers for review  |
| `.codex/config.toml`        | `clients.codex.enabled = true`; flag MCP/server settings for review  |
| `.claude/settings.json`     | `clients.claude.enabled = true`; flag unsafe modes for doctor review |
| `.mcp.json`                 | `clients.claude.enabled = true`; flag non-empty servers for review   |

Import must not import MCP server definitions into `ai-profile.yaml` in MVP.

Canonical generated Markdown marker:

```text
<!-- Generated by Agent Profile Compiler. Do not edit by hand. -->
```

Markdown artifacts with that first line may be reported as generated-looking.
`.codex/config.toml`, `.claude/settings.json`, `.tabnine/mcp_servers.json`, and
`.mcp.json` have no generated marker in the current compiler and must be treated
as user-authored unless future target specs add markers.

## Contracts

- Import analysis must not treat existing artifact text as canonical schema
  fields without user review.
- Import must not copy large raw instruction text into `ai-profile.yaml`.
- Import must not import literal secrets.
- Existing artifacts are not overwritten without explicit write mode and
  diff-before-write.
- `init --import` is the supported command shape. This spec extends
  `002-cli-init.md`; there is no separate `agent-profile import` command in
  Phase 5.
- Import merge order is `(detected stack from 004) + (client enablement and
manual-review signals from this spec) + (guarded defaults from ADR 0002)`.
- Existing `.claude/settings.json` with `defaultMode: "bypassPermissions"` must
  produce an import warning so a later doctor run can enforce `LINT-PERM-004`.
- Existing advanced capability artifacts such as hooks, subagents, plugins, or
  global memories are manual-review signals only in Phase 5. Import must not
  copy them into `ai-profile.yaml` and must not generate matching artifacts
  without later target-specific specs.

## Security Rules

- Do not upload artifact contents.
- Do not print secrets or matched secret-like values.
- Do not read `.env` files.
- Do not mutate files in dry-run.
- Do not install dependencies.

## Acceptance Criteria

- existing supported artifacts are listed in deterministic order
- existing generated-looking files are recognized by header/shape where present
- literal secret-like values produce warnings without echoing values
- import report distinguishes imported signals from manual-review notes
- dry-run does not write
- write uses diff-before-write
- non-MVP client artifacts are ignored with no profile effect

## Tests

- detects existing `AGENTS.md`
- detects existing `CLAUDE.md`
- detects Tabnine guideline directory
- detects supported config files
- secret-like artifact content is redacted from issues
- dry-run no mutation
- deterministic report ordering
- repo containing all supported artifact paths reports them in deterministic
  order
- `.claude/settings.json` with `defaultMode: "bypassPermissions"` produces an
  import warning

## Forward Reference: Subagent Imports

`docs/specs/phase-11/` (Draft, not approved) introduces three new generated
artifact roots: `.claude/agents/`, `.codex/agents/`, and
`.tabnine/agent/agents/`. Existing third-party or manually authored files
under those roots must remain manual-review imports under this spec. Import
must not adopt them into `capabilities.delegation.subagents.agents` until a
later spec defines safe adoption with explicit user confirmation. Doctor
orphan handling for subagents is owned by
`docs/specs/phase-11/005-doctor-subagent-checks.md`
(`LINT-SUBAGENT-006`).
